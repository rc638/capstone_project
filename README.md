### 3주차 (2026년 6월 13일)

**계획 변경**
> 소프트웨어 구현 완료 후 하드웨어 통합 테스트 진행.
> 센서 정확도 문제로 인해 타격 세기 측정 정확도에 한계가 있음을 확인.

#### 하드웨어
- [x] 드럼 패드 제작 완료
- [x] PP-A495 피에조 센서 모듈 연결 및 테스트
- [x] ADC 핀 변경 테스트 (GPIO4 → GPIO1 → GPIO2, GPIO2에서 비교적 안정적인 입력 확인)
- [x] 풀다운 저항 (10kΩ) 추가 시도 → 노이즈에 유의미한 변화 없음
- [x] 패드 레이어 구조 변경 (피에조 위치, 실리콘 완충재 배치 등 다양한 구조 시도)
- [x] 절연테이프를 이용한 신호선 및 피에조 주변 절연 처리

#### 소프트웨어
- [x] 타격 감지 알고리즘 개선 (Threshold 조정, 피크 추적, 순간 변화량 기반 판단 등 다양한 방식 시도)
- [x] 이상치 제거 (MAX_VAL 기준) 및 쿨다운 로직 적용
- [x] Node-RED 대시보드 기능 개선
  - [x] 싱글/더블스트로크 모드 전환 버튼 연결 문제 수정 (ui-template → function 노드 연결)
  - [x] 더블스트로크 막대 가로/세로 표시 문제 수정 (% → px 단위)
  - [x] 타격 후 3초 경과 시 화면 초기화 기능 구현
- [x] 발표 자료 (PPT) 제작

#### 센서 한계 확인
- 피에조 중심을 정확히 타격했을 때는 값 출력 확인
- 하지만 타격 위치에 따른 감도 편차가 크고, 정전용량 노이즈로 인해 약한 타격과 노이즈를 소프트웨어적으로 구분하기 어려움
- 하드웨어 + 소프트웨어 양쪽으로 다양한 해결책을 시도했으나 근본적 해결 불가
- PP-A495 피에조 센서는 타격 감지 자체는 가능하나, 정밀한 세기 측정에는 한계가 있는 센서로 판단
- 향후 FSR 압력 센서 또는 드럼 전용 트리거 회로로 교체 필요
